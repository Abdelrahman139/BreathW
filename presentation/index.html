The Problem

🏥 Not Enough Doctors

There are too many X-rays to check and not enough doctors. Being tired can lead to mistakes.

⏱️ Waiting Too Long

Waiting hours to get the results of an emergency X-ray can be very dangerous or even deadly.

🔀 Different Opinions

Two doctors can look at the exact same X-ray and come to completely different conclusions.

🔒 Lack of Trust in AI

Doctors don't want to use AI programs that just give a final answer without showing the reasons why.

Dataset & Class Balancing

📉

Before (Raw NIH Data)

  • ✗

    112,120 total images across 15 labels.

  • ✗

    Massive Imbalance: 'Healthy' scans represent 53.8% of the data.

  • ✗

    Rare Diseases: Some diseases have fewer than 300 images total.

The Danger: Training on this raw data creates a heavily biased "lazy" AI that just predicts everyone is 'Healthy'.
➡️
⚖️

After (BreathWise Data)

  • ✓

    Strictly filtered to the 5 critical target conditions.

  • ✓

    Perfectly Balanced: Exactly 5,500 images per class.

  • ✓

    Gold Standard Split: 80% Training / 20% Validation.

The Solution: Rare diseases (like Pneumonia) were mathematically upsampled so the AI learns all diseases equally without bias.

End-to-End Data Flow

1

Upload

Doctor securely uploads the X-ray image via the React drag-and-drop interface.

2

API Gateway

C# .NET API receives the file, assigns a secure GUID, and saves it to local storage.

3

AI Trigger

C# Backend triggers a fast internal POST request to the Python FastAPI microservice.

4

Inference & Grad-CAM

DenseNet-121 model predicts diseases while Grad-CAM generates the visual heatmap.

5

Database Write

C# decodes the base64 heatmap payload and writes the final results to SQL Server.

6

Interactive Display

React instantly renders the diagnosis and allows doctors to toggle the AI heatmap overlay.

The AI Model — DenseNet-121

🧠
🧠

How It Works

  • ▶ Connects Everything: Every layer gets information directly from all previous layers.
  • ▶ Learns Step-by-Step: It learns 32 new patterns at a time, slowly building knowledge.
  • ▶ Compresses Data: Uses special layers to shrink the data and keep the model fast.
⚙️
⚙️

How It Decides

  • ▶ Final Output: Takes everything it learned and makes 5 final disease predictions.
  • ▶ Fixing Mistakes: Measures how wrong its guesses are so it can correct them.
  • ▶ Learning Speed: Learns at the perfect speed by looking at 32 X-rays at once.
⚡
⚡

Why Choose It?

1. It's Smaller & Faster

It produces half the data (1024 features) compared to older models like ResNet.

2. It Doesn't Forget

Direct connections stop the AI from forgetting what it learned early on.

3. It Recycles

It reuses old features, so it stays highly accurate while remaining compact.

Training & Results

Model Architecture & Training Flow

  • ▶ Base architecture utilizes pre-trained DenseNet-121 for optimal feature extraction without massive parameter bloat.
  • ▶ Custom classification head trained for multi-label prediction using Binary Cross Entropy (BCEWithLogitsLoss).
  • ▶ Optimized via Adam (LR=1e-4) with automated learning rate decay scheduling on plateaus.
  • ▶ Early stopping enforced at Epoch 5 to completely avoid overfitting, locking in highly generalized weights.

Post-Training Optimizations

🎛️
🎛️

Optimal Thresholds

Default 0.5 cutoffs are suboptimal. We swept 0.0–1.0 to maximize the F1-Score for each specific disease.

Pneumonia0.39
Effusion0.45
Cardiomegaly0.54
Pneumothorax0.39
Clinical Impact: Acute conditions (like Pneumothorax) get lower thresholds, ensuring high sensitivity and fewer missed critical diagnoses.
✨
✨

Test-Time Augmentation

Instead of 1 prediction, the AI runs 3 variations (Original + H-Flip + 10° Rotation) per image and averages the output.

Progressive Accuracy Boost

Baseline (0.5 Cutoff) 89.37%
+ Optimal Thresholds 89.33%
+ Test-Time Augmentation 89.67%
The Result: TTA effectively smooths out visual noise, pushing the model to Grandmaster-level stability.

Explainability — Grad-CAM

Breaking the Black Box

  • Computes gradients of class score vs DenseBlock4 activations
  • Global Average Pooling → importance weights α
  • Weighted sum → ReLU → upsampled to 224×224
  • Mapped to JET color palette, blended α=0.4
  • 3 View Modes: Original | Overlay | Side-by-Side

Security & Patient Privacy

🛡️
🔐

JWT Authentication

Secure 24-hour token expiry, strictly validated on every single API request to prevent unauthorized access.

👥

Role-Based Access

Strict data separation: Doctors have full CRUD access, while Patients are strictly read-only for their own data.

🔑

PBKDF2 Hashing

Passwords secured with HMAC-SHA256, utilizing a unique 128-bit salt per user across 10,000 hash iterations.

🏷️

Data Anonymization

X-rays are saved using GUIDs (e.g. 8c4e2a87.png). Zero patient PII is stored on disk.
